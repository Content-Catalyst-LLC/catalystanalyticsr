# Catalyst Analytics R v0.1.4

## Repository Integrity and Package Contract Repair

This release repairs the v0.1.3 prototype without expanding the model domain.

### Package contract

- Replaces placeholder tests with behavioral tests for demos, exports, glossary records, validation, and carbon-budget integration.
- Removes the duplicate `theme_catalyst()` implementation.
- Consolidates duplicated Roxygen examples and S3 documentation blocks.
- Removes course-specific prototype language from production source and generated documentation.
- Adds shared internal validation for times, states, policies, parameters, flags, strings, and numeric scalars.
- Rejects unknown model parameters and invalid elasticity combinations.
- Records model, integration, scenario, initial-state, policy, and parameter metadata with every simulation.

### Export contract

- Removes stale `kaya` and top-level `params` export assumptions.
- Exports `parameters.csv`, `policy.csv`, and `run_metadata.json` from the canonical run metadata structure.
- Adds sanitized run identifiers, file sizes, and MD5 inventory records.
- Returns the complete written-file inventory from export functions.

### Repository contract

- Moves sample JSON from `data/` to `inst/extdata/`, the correct location for raw package resources.
- Introduces separate, fully constrained schemas for browser scenario inputs and browser exports.
- Synchronizes repository and R package version `0.1.4` through the release manifest.
- Preserves monotonic WordPress plugin versioning at `1.0.1` and records compatibility with repository `0.1.4`.
- Adds static release checks for versions, schemas, placeholder tests, duplicate functions, JavaScript, PHP, and Python tests.
- Expands CI to run the complete repository contract plus `R CMD check`.

### Compatibility boundary

The WordPress browser demo remains a simplified educational implementation. Its export now explicitly records `parity_status: conceptual_only`; it must not be represented as numerically identical to the R engine.

## Release-check repair

The final v0.1.4 package payload also:

- compares deterministic demo trajectories without treating their distinct scenario labels as a model difference;
- uses ASCII-only R source for portable package checks;
- excludes repository-only files and generated Python caches from the built R source package; and
- runs Python contract tests without creating `.pytest_cache`.
