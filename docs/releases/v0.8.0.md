# Catalyst Analytics R v0.8.0

## Calibration, Validation, and Model Governance

This release adds bounded parameter calibration, historical and holdout validation, residual diagnostics, numerical solver and stability evidence, model cards, parameter cards, assumptions, limitations, reviewers, approvals, and governed lifecycle transitions.

### Release boundary

The included deterministic benchmark demonstrates the contract and testing workflow. It is not real-world forecasting evidence. A model can only be marked `validated_for_specified_use` when calibration and validation evidence exist and a reviewer approves a clearly stated use boundary.

### Companion plugin

WordPress companion v1.7.0 provides an educational calibration and holdout workflow, numerical evidence summary, model card, limitations, lifecycle recommendation, and canonical governance JSON export.

## R CMD check repair

The repaired v0.8.0 package recursively converts custom Catalyst S3 records into plain JSON-safe values before export. It also uses explicit scalar extraction for solver and stability comparisons rather than relying on data-frame coercion. Regression tests cover model cards, governance records, calibration and validation objects, numerical evidence, dates, and timestamps.
## R CMD check serialization repair 2

The second same-version repair removes the final unsafe `as.character()` fallback from the recursive JSON sanitizer. Unsupported objects now become plain class/type descriptors, while raw, complex, pairlist, date/time, language, matrix, data-frame, atomic, and Catalyst list records are normalized explicitly. Regression tests cover an unsupported S4 object and special R value types.


## R CMD check repair 3

This same-version repair updates stale export-test version expectations and validates every calibration target before optimization. Targets must reference an existing field under `parameters`, `policy`, or `initial_state`; unsupported or missing targets now fail immediately and consistently.
