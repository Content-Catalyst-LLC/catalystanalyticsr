# Catalyst Analytics R v1.1.0

## Saved Workspaces and Scenario Libraries

- Added persistent multi-project workspaces.
- Added active-project selection and project removal.
- Added reusable scenario and parameter-set libraries.
- Added scenario cloning with independent fingerprints.
- Added reusable policy packages with validated references.
- Added consolidated run history and project comparison.
- Added workspace snapshots and complete state restoration.
- Added canonical workspace JSON import and export.
- Added portable workspace bundles with CSV indexes, Markdown, file hashes, and ZIP output.
- Added WordPress companion v2.1.0.

## R CMD check repair

- Workspace removal now retains `active_project_id` as a named nullable field rather than deleting the field through `$<- NULL`.
- Snapshot restoration and JSON import normalize missing or list-shaped null values before validation.
- Lightweight workspace exports retain an exact `result: null` run field, preventing `$result` from partially matching `result_summary`.
- Regression tests cover null active projects, complete snapshot restoration, and result omission round trips.

## Snapshot fingerprint repair

- Snapshot restoration now returns the fingerprint recorded when the snapshot was created.
- In-memory snapshots retain exact nested R objects rather than flattening them before restoration.
- JSON-imported snapshots preserve semantic identity even when `jsonlite` normalizes internal R container types.
- Restoration-only fingerprint metadata is internal and is not written to canonical workspace JSON.
- Any project, scenario, parameter, policy, or active-project change clears the restored identity and produces a new content fingerprint.
