# Catalyst Analytics R v0.2.0

## Canonical Scenario Contract and Model Registry

Version 0.2.0 converts Catalyst Analytics R from a single-model package into a governed analytical foundation.

## Canonical scenario contract

The new `catalyst_scenario` object records:

- Scenario id, title, and analytical role
- Exact model id and version
- Time definition and values
- Initial state
- Static policy controls
- Parameter overrides
- Constraints such as an emissions budget
- State and flow units
- Geographic and sector scope
- Currency and price year
- Sources, assumptions, and uncertainty records
- Review status
- Creation and migration metadata

The contract uses schema version `1.0.0` and supports validated JSON round trips and stable fingerprints.

## Model registry

The new `catalyst_model` interface declares:

- Required states and default initial state
- Default parameters and policy controls
- Supported integration methods
- State and flow units
- Canonical flow mapping
- Indicator mapping
- Derivative and flow callbacks
- State, policy, and parameter validators
- Model status and intended-use metadata

KH-NC-PA is now registered as model `khncpa` version `1.0.0`.

## Compatibility and migrations

- Legacy R scenario schema `0.1.0` migrates to canonical schema `1.0.0`.
- Browser inputs map to canonical KH-NC-PA scenarios.
- Canonical KH-NC-PA scenarios can map back to browser inputs.
- Browser structure is transferable, but JavaScript and R equations remain numerically distinct.

## Reference fixtures

The release includes:

- A canonical scenario fixture
- A model manifest fixture
- A browser-to-canonical mapping fixture
- A deterministic RK4 numerical trajectory fixture
- A declared numerical tolerance used by R regression tests

## Export governance

Runs created from canonical scenarios now export the complete scenario record, scenario fingerprint, exact model version, and model contract version.

## WordPress demo

The demo plugin advances to `1.1.0`. Downloaded JSON uses browser export schema `1.1.0`, contains a canonical scenario record, and declares `parity_status: mapped_contract`.

## Validation gate

The release contract validates all JSON and schemas, model and scenario fixtures, export provenance, R documentation aliases, JavaScript syntax, PHP syntax, Python repository tests, and the full R package when an R runtime is available.
